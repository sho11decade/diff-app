# 要件定義書
間違い探しを生成するシステムについての要件定義書です。
研究プロジェクトとして構築するため、説明可能であることが求められます。
## 目的
ユーザーが入力した画像から、間違い探しを自動的に生成する研究プロジェクト
## 開発環境
- GPU: NVIDIA Quadro M2000 (4GB VRAM), CUDA: 13.0
- OS: Ubuntu 22.04 LTS Server
- 言語: Python 3.10以上 (環境管理はuvを使用)
- バージョン管理: Git
- フレームワーク: FastAPI (バックエンドAPIの構築)
- ライブラリ: 別途管理
## アーキテクチャ概要
- 前処理
    - 画像のリサイズ（VRAMに収まるサイズに変更）、2値化画像の生成
- セマンティックセグメンテーション（DeepLab v3+）
- 物体の検出と特徴抽出（YOLOv5）
- サリエンシーマップ（顕著性マップ）の生成（DeepGaze, U2Net）
- 間違い生成箇所の選定（セマンティックセグメンテーションとサリエンシーマップを組み合わせて選定）
- 間違いの生成（自然に見えるように、セマンティックセグメンテーションの情報を活用して、物体の位置や特徴を考慮して生成）
    - 低レベル編集: 色、明るさ、テクスチャの変更
    - 幾何的な編集: 物体の位置や形状の変更
    - 高度な編集: 削除、追加、物体の入れ替えなど
- 出力
    - 間違い探し画像の生成
    - 間違いの位置情報の出力（JSON形式など）
    - 答え（間違い箇所に〇を付ける）
## API設計
- エンドポイント: `/generate`
- メソッド: POST
- 入力: 画像ファイル（JPEG, PNGなど）、オプションパラメータ（間違いの数、難易度など）
- 出力: 間違い探し画像、間違いの位置情報、答え画像
## 性能要件
- 生成時間: 1枚あたり30秒以内を目標
- 間違いの自然さ: 人間が見ても自然に見えること
- 間違いの多様性: 同じ画像から複数の異なる間違い探しを生成できること
## セキュリティ要件
- 入力画像のサイズ制限（例: 5MB以下）
- APIへのアクセス制限（例: APIキーによる認証、レートリミット）
## テスト要件
pytestを使用して、以下のテストを実施
- 単体テスト: 各モジュールの機能を個別にテスト
- 統合テスト: システム全体の動作をテスト
- 性能テスト: 生成時間やリソース使用量を測定
- ユーザビリティテスト: 生成された間違い探しの自然さや多様性を評価
## ドキュメント要件
- コードドキュメント: 各関数やクラスに対するドキュメントコメントの記述
- APIドキュメント: FastAPIの自動生成機能を活用してAPIドキュメントを提供
- 開発ドキュメント: 開発環境のセットアップ手順、コード構成、使用方法などをREADME.mdに記載
- 研究ドキュメント: 研究の背景、手法、実験結果などをまとめたドキュメントを別途作成
