# 研究成果発表 スライドトピック整理

## 1. 発表の狙い
- 目的: 自然な間違い探し画像を、説明可能かつ再現可能に生成する研究開発の成果を示す。
- 伝える価値: 生成品質だけでなく、なぜその差分を作ったかを追跡できる点を強みとして提示する。
- 成果の軸: 品質改善、再現性、評価可能性、デモ可能性。

## 2. 想定オーディエンス
- 画像生成・画像処理の研究者
- システム実装を重視するエンジニア
- プロダクト化視点の技術企画

## 3. コアメッセージ
- メッセージ1: 差分生成を単発処理ではなく、評価可能な研究パイプラインとして実装した。
- メッセージ2: 自然さ評価と品質ゲートにより、不自然差分を段階的に低減した。
- メッセージ3: trace_id と seed で再現性を確保し、実験比較が可能な基盤を整備した。
- メッセージ4: デモUIにより、定量評価と人間知覚の両面で成果を提示できる。

## 9. 先行研究（SAGUS）との比較

### 9.1 比較対象
- 対象論文: [docs/SAGUS論文.txt](docs/SAGUS論文.txt)
- 本システム側参照: [README.md](README.md), [docs/研究アーキテクチャ提案.md](docs/研究アーキテクチャ提案.md), [docs/自然性分析レポート.md](docs/自然性分析レポート.md)

### 9.2 優位性として提示できる点
- 研究運用性: seed と trace_id による再現、差分カード保存、アブレーション自動化により、比較実験を反復しやすい。
- 品質改善ループ: 編集 → 自然さ評価 → 品質ゲート → フォールバックの閉ループで、不自然差分を段階的に低減できる。
- 領域選択の拡張性: 背景回避スコアに加え、セグメンテーション統合で前景比率を利用できるため、背景誤編集の抑制余地が大きい。
- API中心設計: FastAPIベースのため、デモ用途だけでなく実験バッチや外部連携に展開しやすい。

### 9.3 公正に示すべき注意点
- SAGUSは被験者評価（11名）を明示しており、主観評価の報告が明確。
- 本システムは内部指標・アブレーションが強みで、被験者規模評価は今後の拡張課題。
- 発表時は「全面的な上位互換」ではなく、「再現性と改善運用性で優位、ユーザ主観評価は今後強化」と表現する。

### 9.4 スライド用の主張文（短文）
- 本研究はSAGUSの顕著性利用による自動生成の着想を継承しつつ、品質ゲートと自己改善ループ、実験トレース、アブレーション自動化を統合することで、自然さ改善の反復可能性と研究再現性を向上させた。
